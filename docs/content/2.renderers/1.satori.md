Nuxt OG Image uses [Satori](https://github.com/vercel/satori) as the default renderer. It converts Vue components to SVG, then rasterizes to PNG.

## Installation

::code-group

```sh [pnpm]
# Node.js
pnpm i -D satori @resvg/resvg-js

# Edge runtimes (Cloudflare, Vercel Edge, etc.)
pnpm i -D satori @resvg/resvg-wasm
```

```bash [yarn]
# Node.js
yarn add -D satori @resvg/resvg-js

# Edge runtimes (Cloudflare, Vercel Edge, etc.)
yarn add -D satori @resvg/resvg-wasm
```

```bash [npm]
# Node.js
npm install -D satori @resvg/resvg-js

# Edge runtimes (Cloudflare, Vercel Edge, etc.)
npm install -D satori @resvg/resvg-wasm
```

::

## Usage

It's not necessary to manually define the renderer as `satori` unless you changed the default.

::code-group

```ts [Set Default]
export default defineNuxtConfig({
  // this is not necessary as satori is already the default
  ogImage: {
    defaults: {
      renderer: 'satori'
    }
  }
})
```

```ts [defineOgImageComponent]
defineOgImageComponent('MyOgImage', {
  renderer: 'satori' // only when the default has been changed
})
```

::

## Pros

- Fast rendering
- Works on all environments including edge runtimes
- Good CSS support (flexbox, fonts, colors, gradients)
- 6 emoji families supported
- Google Fonts integration

## Cons

- Some CSS limitations (see below)
- Requires learning Satori's constraints

## Limitations

It's important to familiarize yourself with Satori before you make more complex templates.

::warning
Satori has CSS limitations. For complex templates, consider the [Chromium renderer](/docs/og-image/renderers/chromium).
::

If you find Satori too difficult to work with, you can use [Takumi](/docs/og-image/renderers/takumi) for simpler templates or [Chromium](/docs/og-image/renderers/chromium) for full CSS support.

To learn more about the limitations and working around them, see:
- [Styling](/docs/og-image/guides/styling)
- [Icons and Images](/docs/og-image/guides/icons-and-images)
