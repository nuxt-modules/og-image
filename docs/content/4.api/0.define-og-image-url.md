## Introduction

The `defineOgImageUrl()`{lang="ts"} composable sets a pre-prepared image as the og:image for the current page, bypassing image generation entirely.

Use this when you already have an image (in your `public` directory, on a CDN, or any external URL) and don't need the module to render one.

## Syntax

```ts
defineOgImageUrl(url, options?)
```

- `url`: The image URL - absolute or relative. Supports reactive values.
- `options`: Optional `width`, `height`, `alt`, and other Open Graph image properties.

## Usage

### Basic

```vue
<script setup>
// Public directory image
defineOgImageUrl('/images/og-home.png')

// External URL
defineOgImageUrl('https://cdn.example.com/og/about.png')
</script>
```

### With Options

```vue
<script setup>
defineOgImageUrl('https://cdn.example.com/og.png', {
  width: 1200,
  height: 630,
  alt: 'My page description',
})
</script>
```

### Reactive URL

The URL can be reactive using a ref or getter:

```vue
<script setup>
const ogUrl = ref('/images/default-og.png')

defineOgImageUrl(ogUrl)

// or with a getter
defineOgImageUrl(() => `/images/og-${locale.value}.png`)
</script>
```

### Override a Generated Image

A common pattern is generating OG images globally but overriding specific pages with a pre-prepared image:

```vue
<!-- app.vue -->
<script setup>
// Generate OG images for all pages by default
defineOgImage('NuxtSeo', { title: 'My Site' })
</script>
```

```vue
<!-- pages/landing.vue -->
<script setup>
// Override with a hand-crafted image for the landing page
defineOgImageUrl('/images/landing-og.png', {
  width: 1200,
  height: 630,
  alt: 'Welcome to My Site',
})
</script>
```

## Props

### `width`

- Type: `number`{lang="ts"}
- Default: `undefined`{lang="ts"}

The width of the image in pixels.

### `height`

- Type: `number`{lang="ts"}
- Default: `undefined`{lang="ts"}

The height of the image in pixels.

### `alt`

- Type: `string`{lang="ts"}
- Default: `undefined`{lang="ts"}

Alt text for the image. Recommended for accessibility.

## Migration from v5

In v5, you could pass a `url` property to `defineOgImage()`{lang="ts"}:

```ts
// v5 (no longer works)
defineOgImage({ url: '/my-image.png' })

// v6
defineOgImageUrl('/my-image.png')
```

See the [migration guide](/docs/og-image/migration-guide/v6#object-syntax-with-url-replaced-by-defineogimageur) for more details.
