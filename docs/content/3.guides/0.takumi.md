Takumi is an alternative renderer built on Rust that directly rasterizes to PNG/JPEG without an SVG intermediate step.

It's designed as a faster alternative to Satori for simple templates.

::code-group

```ts [Set Default]
export default defineNuxtConfig({
  ogImage: {
    defaults: {
      renderer: 'takumi'
    }
  }
})
```

```ts [defineOgImage]
defineOgImage({
  renderer: 'takumi'
})
```

```ts [defineOgImageComponent]
defineOgImageComponent('MyOgImage', {
  renderer: 'takumi'
})
```

::

## Installation

Takumi requires installing the `@takumi-rs/core` package (or `@takumi-rs/wasm` for edge runtimes).

::code-group

```sh [pnpm]
pnpm i -D @takumi-rs/core @takumi-rs/wasm
```

```bash [yarn]
yarn add -D @takumi-rs/core @takumi-rs/wasm
```

```bash [npm]
npm install -D @takumi-rs/core @takumi-rs/wasm
```

::

## Pros

- **2-10x faster** than Satori+Resvg for simple templates
- Direct PNG/JPEG rasterization (no SVG intermediate)
- Native Tailwind CSS support via `tw` prop
- Works on Node.js and edge runtimes (WASM)

## Cons

- Limited CSS support compared to Satori
- No gradient backgrounds
- Opacity color syntax not fully supported
- Some flexbox features missing

## Limitations

Takumi has more CSS limitations than Satori. It works best with simple, solid-color templates.

### Supported Features

| Feature | Status |
|---------|--------|
| Solid background colors | ✅ |
| Basic flexbox (`flex`, `items-center`, `justify-center`) | ✅ |
| Text colors (solid) | ✅ |
| Font sizes and weights | ✅ |
| Rounded corners | ✅ |
| Emoji | ✅ |
| Images | ✅ |

### Unsupported Features

| Feature | Status |
|---------|--------|
| Gradient backgrounds (`bg-gradient-to-*`) | ❌ |
| Opacity colors (`text-white/60`) | ❌ |
| `justify-between`, `space-between` | ❌ |
| `flex-1`, `flex-grow` | ❌ |
| Complex CSS transforms | ❌ |

::warning
For complex templates with gradients or opacity, use the [Satori renderer](/docs/og-image/guides/satori) instead.
::

## When to Use Takumi

Takumi is ideal for:
- Simple templates with solid colors
- High-throughput scenarios where speed matters
- Edge runtime deployments where bundle size is important

## Example Template

A Takumi-compatible template using only supported features:

```vue
<script setup lang="ts">
defineProps<{
  title: string
}>()
</script>

<template>
  <div class="w-full h-full flex items-center justify-center bg-gray-900 text-white">
    <h1 class="text-6xl font-bold">
      {{ title }}
    </h1>
  </div>
</template>
```

## Runtime Compatibility

Takumi supports both Node.js and WASM runtimes:

| Environment | Binding |
|-------------|---------|
| Node.js | `@takumi-rs/core` |
| Cloudflare Workers | `@takumi-rs/wasm` |
| Vercel Edge | `@takumi-rs/wasm` |
| Netlify Edge | `@takumi-rs/wasm` |

The module automatically selects the appropriate binding based on your deployment target.
