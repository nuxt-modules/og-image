Nuxt OG Image supports modern CSS styling via Tailwind v4, UnoCSS, and standard CSS. The level of CSS support depends on the renderer you choose.

## Layout Support

| Feature | [Takumi](/docs/og-image/renderers/takumi) | [Satori](/docs/og-image/renderers/satori) | [Browser](/docs/og-image/renderers/browser) |
|---------|--------|--------|----------|
| **Flexbox** | ✅ | ✅ | ✅ |
| **CSS Grid** | ✅ | ❌ | ✅ |
| **Block/Inline** | ✅ | ❌ | ✅ |
| **Position Absolute** | ✅ | ✅ | ✅ |

### Satori Constraints

When using the [Satori](/docs/og-image/renderers/satori) renderer, everything is treated as a flexbox with a `flex-direction` of `column` by default. It does not support `display: grid`, `display: block`, or `display: inline`.

### Takumi & Browser

The recommended [Takumi](/docs/og-image/renderers/takumi) renderer and the [Browser](/docs/og-image/renderers/browser) renderer have much broader CSS support, including CSS Grid and standard block/inline layouts.

## Build-time Transformations

To ensure your templates look great across all renderers, Nuxt OG Image performs several "automagic" CSS transformations at build time.

### CSS Downleveling & Polyfills

The module uses [Lightning CSS](https://lightningcss.dev/) to process your styles and ensure compatibility with simpler renderers like Satori:

- **Color Downleveling:** Modern colors like `oklch()`, `lab()`, or `p3` are automatically downleveled to standard hex or `rgba()` values.
- **Logical Properties:** Properties like `padding-inline` or `margin-block` are expanded to their physical counterparts (`padding-left`, `padding-right`, etc.).
- **Individual Transforms:** Properties like `translate`, `rotate`, and `scale` are converted into a single `transform` shorthand.
- **Color Mix:** `color-mix()` functions are evaluated and replaced with static color values.
- **Calc Evaluation:** Simple `calc()` expressions are evaluated where possible.

### Unsupported Classes

Some utility classes (like animations, transitions, or z-index) that aren't supported by static OG image renderers are automatically filtered out during the build process to prevent rendering errors.

## Tailwind CSS & UnoCSS

Out-of-the-box and without any extra configuration, you can use Tailwind/UnoCSS classes in your templates.

```html
<div class="bg-blue-500 text-white p-12 flex flex-col items-center justify-center">
  <h1 class="text-6xl font-bold shadow-xl">Hello World</h1>
</div>
```

The module extracts classes from your OG components at build time and compiles them using your project's CSS engine (Tailwind v4 or UnoCSS).

### Theme Customizations

The module automatically resolves custom theme values from your CSS. For Tailwind v4, this includes variables defined in your `@theme` block:

```css [assets/css/main.css]
@import "tailwindcss";

@theme {
  --color-brand: #ff6600;
}
```

```html
<template>
  <div class="bg-brand text-white">
    Custom Theme Classes!
  </div>
</template>
```

### Auto-detection

The module auto-detects your CSS framework:
- **Tailwind v4:** Detected if `tailwindcss` is installed and an `@import "tailwindcss"` is found in your CSS.
- **UnoCSS:** Detected if `@unocss/nuxt` is enabled.
- **Nuxt UI v3:** Colors and theme variables are automatically imported.

## Inline Styles

For dynamic values that change based on props, use inline styles via the `:style` binding.

```html
<template>
  <div :style="{ backgroundColor: themeColor }" class="w-full h-full flex items-center justify-center">
    <h1 style="color: white; font-size: 80px;">
      {{ title }}
    </h1>
  </div>
</template>
```

## `<style>` tag support

You can use scoped or global `<style>` tags within your components. The module will process these and apply them to the rendered image.

::note
Standard CSS selectors (like `.my-class`) work across all renderers. However, Satori has limited support for advanced selectors like `:nth-child` or complex combinators.
::
